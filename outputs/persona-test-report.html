1. 개요

상권AI 챗봇의 기능 개발이 완료된 시점에서, 실제 업무 맥락을 가진 두 명의 가상 실무자가 도구를 처음 사용하며 각자의 프로젝트 목표를 달성할 수 있는지 검증했다. 두 페르소나 모두 도구 안에 어떤 데이터가 있는지 모르는 상태에서 시작하여, 대화를 통해 기능을 탐색하고 한계를 발견하는 탐색적 테스트 방식이다.

총 24턴의 질문-응답을 실행했으며, 각 턴에 대해 5개 차원(정확성·이해도·유용성·표현력·연결성)을 1~5점으로 평가했다. 테스트는 Gemini API 키 없이 로컬 모드로 진행되어, 파이프라인의 로컬 폴백 경로만 검증했다.

5. 발견 이슈 & 개선 제안

P0
merge/sgg 결과 후 follow-up 컨텍스트 전면 실패
법정동 merge(둔산동, 관저동, 유성온천) 또는 구(sgg) 단위 분석 후, "유동인구는?", "편의점은?", "비슷한 상권 있어?" 같은 follow-up이 모두 "지역이나 업종이 부족합니다"로 실패. 단일 행정동 후에는 정상 작동(B12).
재현: A11, B2, B6, B8, B10 — 5/24턴(21%) 실패
개선 제안: conversation-state에서 merge 결과의 대표 행정동(예: 가장 큰 동)과 sgg를 activeDistrict/activeSgg로 저장. follow-up 시 이를 기본값으로 사용.
P1
"노은동" merge 미트리거 (법정동 매핑 불완전)
"둔산동"은 locationAlias를 통해 둔산1+2+3동으로 merge되지만, "노은동"은 노은1동만 선택됨. legalDongToAdminDong 또는 locationAliases에 "노은동" → [노은1동, 노은2동, 노은3동] 매핑이 누락된 것으로 추정.
재현: B11
개선 제안: matching-dictionaries.json의 legalDongToAdminDong에 "노은동" 매핑 추가. 유사 패턴(갈마동, 도마동, 유천동 등)도 일괄 점검.
P2
merge 평균 매출이 단순 산술평균
둔산동 merge: 단순평균 1,539만원 vs 업소수 가중평균 1,814만원 (18% 차이). 업소 20개인 둔산3동(894만원)과 121개인 둔산2동(2,077만원)이 동일 비중.
개선 제안: 업소 수 가중평균으로 변경하거나, "가중/단순" 레이블을 표기하여 해석 오류 방지.
P2
"작년 대비" 질문에 기존 매출 카드 중복 표시
A3("매출은?")과 A4("작년 대비 어때?")가 동일 카드. YoY가 이미 포함되어 있어 추세 전용 뷰(12개월 차트 강조, 분기별 비교 등)가 별도로 제공되지 않음.
개선 제안: "작년", "추세", "변화" 키워드에 대해 trend 뷰(12개월 차트 풀사이즈 + 분기 하이라이트)를 우선 표시.
P2
데이터 없음 시 대안 제안 부재
"자양동 네일 매출" → "데이터가 없습니다"로만 안내. 자양동에서 조회 가능한 업종이나, 네일이 있는 인근 동을 제안하지 않음.
